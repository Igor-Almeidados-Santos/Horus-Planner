# Regras de Negócio Críticas

## 1. Todo plano deve ter versão
Quando o agente ou o usuário alterar a estrutura de um plano de forma importante, gere nova versão.

## 2. Tarefa não deve ser apagada fisicamente
Use soft delete ou arquivamento, para preservar histórico.

## 3. Execução deve guardar tempo real
Sem isso, não existe melhoria real do sistema.

## 4. Histórico deve ser preservado
O agente precisa ler padrões passados.

## 5. O usuário pode ajustar manualmente
O sistema não pode ser totalmente rígido.
