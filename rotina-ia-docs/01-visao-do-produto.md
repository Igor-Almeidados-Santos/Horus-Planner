# Visão do Produto

O projeto deve ser tratado como uma **plataforma de execução de rotina assistida por IA**, e não apenas como uma API com interface web.

## Proposta central

O sistema opera em um ciclo contínuo:

**planejar → estruturar → executar → medir → ajustar**

## Fluxo macro

1. O agente GPT conversa com o usuário.
2. O agente transforma a intenção em um plano estruturado.
3. A API recebe o plano, valida, padroniza, versiona e grava.
4. A interface web apresenta rotinas, tarefas, metas, métricas, prioridades, bloqueios, progresso e ajustes.
5. O sistema devolve ao agente dados consolidados para orientar o usuário com mais precisão.

## Objetivo de negócio

Criar um sistema operacional de rotina pessoal com foco em:

- clareza de execução
- mensuração real
- adaptação contínua
- apoio analítico ao usuário
